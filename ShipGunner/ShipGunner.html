Ship Gunner

Overview

Ship Gunner will be a Jack computer game based on the classic board game, Battleship. The game is played on grids for each opponent. Players arrange ships on their grids. Turns are taken to call out grid locations. The player's opponent announces whether the ship has been hit, missed, or sunk. When all the squares of an ship have been hit, the ship is sunk. The player who loses all their ship loses the game.

Specification

The ship pieces shall be defined as follows. The size of the ship is the number of grid squares.

The game shall be played on a 10x10 matrix. The ships are placed horizontally or vertically on the grid. Ship location, hit, miss and sunk ships shall be denoted by coloring the square background.

Gameplay shall be alternating rounds where the player selects a grid location to ping. This shall be accomplished with side by side grid showing the player's grid, and the explored area of the opponent's grid.

Implementation

1. Jack programming language will be used for the implementation.

2. Grid structure shall be stored in a 10x10 grid matrix. Four matrixes shall be required:

Array grid_Player
Array grid_Player_Opponent
Array grid_Opponent
Array grid_Opponent_Player

3. Computer opponent requires exploratory search algorithm with randomization routines. Initial pings should be broad early in the game. Late game the routine should target likely locations for hidden ships.

When computer opponent successfully makes a hit:
1) Routine changes to radial search until next hit is found.
2) If ship is not sunk, proceed along horizontal or vertical line.
2a) If ping is a miss but ship is not sunk, make next attack in the opposite end.
3) Continue until ship is sunk.

4. End condition is reached when Player or Computer sinks all opponent's ships
